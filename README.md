# Discord Odysseia-Count Bot

在 Discord 服务器内提供一套结构化的“投诉 → 管理审核 → 开庭 → 回合制辩诉（自主发言）→ 管理裁决 → 一键归档并删除”流程，减少争吵与信息散落，并保留可追溯的案件记录。

> 说明：本项目是服务器治理工具，不构成任何法律建议。

---

## 功能概览

- **中文 Slash 指令 UX**：以 `/类脑大法庭 ...` 形式使用。
- **立案申请（Modal 表单）**：投诉人提交案件说明、违反规则、证据等。
- **管理审核面板**：管理在审核频道完成通过/驳回/要求补充材料。
- **Ticket 风格“每案一个频道”**：审核通过后创建案件频道（不是 Forum 线程），避免跨帖权限泄露。
- **公开 / 私密模式**：
  - 申请时可选公开/私密；管理可在审核时覆盖。
  - 公开案件可配置“观众身份组”只读围观。
- **自主发言的回合制辩诉（重点）**：
  - 双方默认不能随意发言。
  - 轮到某一方时，由该方点击 **“获取本轮发言权”**，获得临时发言权限。
  - **每轮限制：10 分钟 或 10 条消息（先到先结束）**。
  - 可手动点击 **“结束本轮发言”** 提前结束。
  - 非当前发言者（且非管理）发言会被自动删除。
  - 管理可使用 **“管理强制结束/推进”** 兜底推进流程。
- **三辩后“是否继续辩诉”机制**：
  - 第 3 轮结束后进入投票面板：**希望继续辩诉 / 希望结束辩诉**。
  - **双方都同意继续** 才进入第 4 轮；任意一方选择结束则进入裁决。
  - 提供 **管理强制结束**，防止长期无人点击导致流程卡死。
- **管理裁决（仅管理可操作）**：
  - 按钮：**成立并说明 / 不成立并说明**。
  - 必须填写说明理由（Modal）。
- **结案归档与删除**：
  - 结案或撤诉后，管理可点击 **“归档并删除”**。
  - 导出 **HTML**：
    - **图片优先**：尽量内嵌到 HTML（或打包 ZIP）。
    - 外部链接：仅记录 URL。
    - 非图片附件：仅记录 URL。
  - 归档文件发送到“归档频道”（仅管理可见），随后删除案件频道。
- **撤诉（投诉人）二次确认**：撤诉需要输入口令确认，撤诉后同样可归档并删除频道。

## 指令列表

- `/类脑大法庭 申请开庭`
  - 选择：被投诉人
  - 选择：开庭模式（公开/私密）
  - 可选：证据链接、证据附件（最多 3 个）
  - 随后会弹出表单填写案件说明等信息

- `/类脑大法庭 补充证据`
  - 可选：案件编号（若在案件频道内使用可不填）
  - 可选：证据链接、证据附件（最多 3 个）
  - 随后会弹出表单填写证据说明

- `/类脑大法庭 设置`（仅管理）
  - 配置管理身份组、审核频道、庭审分类、裁决频道、归档频道、观众身份组（可选）、审计频道（可选）

- `/类脑大法庭 查看设置`
  - 查看当前服务器的配置

## Discord 侧准备

### 1) 创建并邀请 Bot
在 https://discord.com/developers 创建应用并添加 Bot，然后邀请进服务器。

建议邀请时勾选：
- Scopes：`bot` + `applications.commands`
- Permissions（建议至少具备）：
  - View Channels / Read Message History
  - Send Messages / Embed Links / Attach Files
  - **Manage Channels**（创建/删除案件频道）
  - **Manage Messages**（删除非当前发言者的插话消息）

### 2) 必须开启的 Privileged Intents
在 Bot 设置中开启：
- **Server Members Intent**
- **Message Content Intent**（用于发言条数统计、纪律兜底与 HTML 归档导出）

## 服务器资源建议

建议准备以下资源（并在 `/类脑大法庭 设置` 中选择）：
- **管理审核频道**：发布案件卡片与审核按钮
- **裁决频道（仅管理可见）**：发布裁决面板
- **庭审分类（Category）**：所有案件频道创建在此分类下
- **归档频道（仅管理可见）**：归档 HTML/ZIP 文件会发送到这里
- （可选）**审计日志频道**：关键操作额外留档
- （可选）**观众身份组**：用于公开案件的只读围观

## 本地运行

### 1) 安装依赖

```powershell
pip install -r requirements.txt
```

### 2) 配置环境变量
复制 `.env.example` 为 `.env` 并填写：
- `DISCORD_TOKEN`：你的 Bot Token
- `COMMAND_GUILD_ID`：开发期建议填写你的服务器 ID（指令同步更快，可选）
- `DB_PATH`：SQLite 数据库路径（可选，默认 `data/court.db`）

> 频道/分类/身份组 **不需要** 写入 `.env`，统一在 Discord 内使用 `/类脑大法庭 设置` 配置。

### 3) 启动

```powershell
python main.py
```

## 典型使用流程（从 0 到归档）

1. 管理先执行：`/类脑大法庭 设置`
2. 投诉人执行：`/类脑大法庭 申请开庭`
3. 管理在审核频道处理案件：通过后自动创建案件频道并发布庭审控制面板
4. 庭审回合制辩诉：当事人按回合点击 **获取本轮发言权** → 在窗口内自行发言
5. 第 3 轮结束后进入“是否继续辩诉”投票（双方同意才进入第 4 轮）
6. 进入裁决：管理在裁决频道选择“成立并说明 / 不成立并说明”并填写理由
7. 结案后在案件频道点击 **归档并删除**：发送归档文件到归档频道并删除案件频道

## 常见问题

- **按钮点了没反应？**
  - 检查 Bot 是否在线、是否具备相应频道的权限。
  - 若刚重启 Bot，等待其完成启动并恢复交互组件。

- **回合发言限制不生效 / 无法统计消息？**
  - 确认已开启 **Message Content Intent**。
  - 确认 Bot 拥有 **Manage Messages** 权限。

- **归档失败或发不出文件？**
  - 确认已在设置中指定“归档频道”。
  - 确认 Bot 在归档频道拥有发送消息与发送文件权限。
