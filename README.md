# Discord Odysseia-Court Bot

在 Discord 服务器内提供一套结构化的“投诉 → 管理审核 → 开始议诉 → 回合制议诉（自主发言）→ 管理裁决 → 一键归档并删除”流程，减少争吵与信息散落，并保留可追溯的议诉记录。

> 说明：本项目是服务器治理工具，不构成任何法律建议。

---

## 功能概览

- **中文 / 指令**：以 `/议诉 ...` 形式使用。
- **议诉申请（Modal 表单）**：投诉人提交申请说明、违反规则、证据等。
- **议诉申请入口面板**：管理可发布带 **“提交议诉申请”** 按钮的入口，并自定义议诉区说明；入口会提醒需要附件时改用 `/议诉 申请`。
- **管理审核面板**：管理在审核频道完成通过/驳回/要求补充材料。
- **Ticket 风格“每个议诉一个频道”**：审核通过后创建议诉频道（不是 Forum 线程），避免跨帖权限泄露。
- **公开 / 私密模式**：
  - 申请时可选公开/私密；管理可在审核时覆盖。
  - 公开议诉可配置“观众身份组”只读围观。
- **自主发言的回合制议诉（重点）**：
  - 双方默认不能随意发言。
  - 轮到某一方时，由该方点击 **“获取本轮发言权”**，获得临时发言权限。
  - **每轮限制：最多 10 条消息，不设时间限制**。
  - 可手动点击 **“结束本轮发言”** 提前结束，或达到条数上限自动结束。
  - 非当前发言者（且非管理）发言会被自动删除。
  - 管理可使用 **“管理强制结束/推进”** 兜底推进流程。
- **三辩后“是否继续议诉”机制**：
  - 第 3 轮结束后进入投票面板：**希望继续议诉 / 希望结束议诉**。
  - **双方都同意继续** 才进入第 4 轮；任意一方选择结束则进入裁决。
  - 提供 **管理强制结束**，防止长期无人点击导致流程卡死。
- **管理裁决（仅管理可操作）**：
  - 按钮：**成立并说明 / 不成立并说明**。
  - 必须填写说明理由（Modal）。
- **结案归档与删除**：
  - 结案或撤诉后，管理可点击 **“归档并删除”**。
  - 导出 **HTML**：
    - **图片优先**：尽量内嵌到 HTML（或打包 ZIP）。
    - 外部链接：仅记录 URL。
    - 非图片附件：仅记录 URL。
  - 归档文件发送到“归档频道”（仅管理可见），随后删除议诉频道。
- **撤诉（投诉人）二次确认**：撤诉需要输入口令确认，撤诉后同样可归档并删除频道。
- **监察组模块（`/监察`）**：独立 Cog 与独立数据库 `data/inspection.sqlite`，用于候补管理、监察案件、3/5/7 单数组抽取、临时讨论、匿名投票与结果公示。
- **监察归档**：管理员可使用 `/监察 归档` 将监察临时讨论频道导出为 HTML/ZIP：
  - 默认 **仅归档**，原临时讨论频道继续保留。
  - 可选 **归档并锁定频道**，保留频道但移除临时监察成员查看权限，使频道仅服务器管理/已有管理覆盖可见。
  - 可选 **归档并删除频道**，导出成功后删除原临时讨论频道。
  - 归档摘要只包含匿名投票统计，不导出“谁投了什么”。
  - 可在 `/监察 设置` 配置可选归档频道；未配置时 `/监察 归档` 会回退发送到 admin 通知频道。

## 指令列表

- `/议诉 申请`
  - 选择：被投诉人
  - 选择：议诉模式（公开/私密）
  - 可选：证据链接、证据附件（最多 3 个）
  - 随后会弹出表单填写申请说明等信息

- `/议诉 补充证据`
  - 可选：议诉编号（可在任意频道填写编号使用；若在对应议诉频道内使用可不填）
  - 可选：证据链接、证据附件（最多 3 个）
  - 随后会弹出表单填写证据说明
  - 适用于提交后补充附件、管理要求补充材料、或议诉进行中追加证据

- `/议诉 设置`（仅管理）
  - 配置管理身份组、审核频道、议诉分类、裁决频道、归档频道、观众身份组（可选）、审计频道（可选）

- `/议诉 入口`（仅管理组）
  - 在当前频道发布带 **“提交议诉申请”** 按钮的入口面板
  - 可选填写“详细说明”，用于自定义议诉区用法；面板会提醒：若需要上传附件，请使用 `/议诉 申请`

- `/议诉 查看设置`
  - 查看当前服务器的配置

- `/监察 设置`（仅服务器 owner 或 Administrator）
  - 配置监察候补身份组、admin 通知频道、临时讨论频道分类、裁决公示频道、留任确认周期天数，以及可选归档频道
- `/监察 成员管理`、`/监察 启动监察`、`/监察 案件管理`、`/监察 投票面板`
  - 管理监察候补、创建案件、Ban/抽取/补抽/取消、在临时讨论频道发起匿名投票
- `/监察 归档`（仅服务器 owner 或 Administrator）
  - 对已公示裁决或已取消的监察案件导出 HTML/ZIP；处理方式可选：仅归档、归档并锁定频道、归档并删除频道

## Discord 侧准备

### 1) 创建并邀请 Bot
在 https://discord.com/developers 创建应用并添加 Bot，然后邀请进服务器。

建议邀请时勾选：
- Scopes：`bot` + `applications.commands`
- Permissions（建议至少具备）：
  - View Channels / Read Message History
  - Send Messages / Embed Links / Attach Files
  - **Manage Channels**（创建/删除议诉频道）
  - **Manage Messages**（删除非当前发言者的插话消息）

### 2) 必须开启的 Privileged Intents
在 Bot 设置中开启：
- **Server Members Intent**
- **Message Content Intent**（用于发言条数统计、纪律兜底与 HTML 归档导出）

## 服务器资源建议

建议准备以下资源（并在 `/议诉 设置` 中选择）：
- **管理审核频道**：发布议诉卡片与审核按钮
- **裁决频道（仅管理可见）**：发布裁决面板
- **议诉分类（Category）**：所有议诉频道创建在此分类下
- **归档频道（仅管理可见）**：归档 HTML/ZIP 文件会发送到这里
- （可选）**审计日志频道**：关键操作额外留档
- （可选）**观众身份组**：用于公开议诉的只读围观

## 本地运行

### 1) 安装依赖

```powershell
pip install -r requirements.txt
```

### 2) 配置环境变量
复制 `.env.example` 为 `.env` 并填写：
- `DISCORD_TOKEN`：你的 Bot Token
- `COMMAND_GUILD_IDS`：建议填写一个或多个服务器 ID，用英文逗号/空格/分号分隔；填写后会按服务器快速同步指令，重启后通常很快可见（可选）
- `COMMAND_GUILD_ID`：旧配置兼容项，只填一个服务器时仍可使用；若同时填写，会与 `COMMAND_GUILD_IDS` 合并去重
- `DB_PATH`：SQLite 数据库路径（可选，默认 `data/court.db`）

> 频道/分类/身份组 **不需要** 写入 `.env`，统一在 Discord 内使用 `/议诉 设置` 与 `/监察 设置` 配置。

### 3) 启动

```powershell
python main.py
```

## 典型使用流程（从 0 到归档）

1. 管理先执行：`/议诉 设置`
2. 管理可在议诉区执行：`/议诉 入口`，发布入口面板
3. 投诉人点击入口面板的 **提交议诉申请**，或执行：`/议诉 申请`（需要上传附件时必须使用 / 指令）
4. 管理在审核频道处理议诉：通过后自动创建议诉频道并发布议诉控制面板
5. 回合制议诉：当事人按回合点击 **获取本轮发言权** → 在窗口内自行发言
6. 第 3 轮结束后进入“是否继续议诉”投票（双方同意才进入第 4 轮）
7. 进入裁决：管理在裁决频道选择“成立并说明 / 不成立并说明”并填写理由
8. 结案后在议诉频道点击 **归档并删除**：发送归档文件到归档频道并删除议诉频道

## 监察典型使用流程

1. 管理先执行：`/监察 设置`，其中“归档频道”为可选项，不影响核心监察命令启用。
2. 管理执行：`/监察 成员管理` 添加或维护监察候补。
3. 管理执行：`/监察 启动监察` 创建案件并向候补发送 DM 响应按钮。
4. 响应结束后按规则进入 Ban/抽取阶段，最终临时监察组人数只会是 3 / 5 / 7。
5. 临时监察成员在私密讨论频道讨论，管理员或本案成员执行 `/监察 投票面板` 发起匿名投票。
6. 投票结束后自动公示匿名统计结果：诉求合理 / 诉求不合理 / 未投票人数。
7. 管理可执行 `/监察 归档` 导出 HTML/ZIP；默认仅归档并保留讨论频道，也可选择锁定为仅管理可见或删除。

## 常见问题

- **按钮点了没反应？**
  - 检查 Bot 是否在线、是否具备相应频道的权限。
  - 若刚重启 Bot，等待其完成启动并恢复交互组件。

- **回合发言限制不生效 / 无法统计消息？**
  - 确认已开启 **Message Content Intent**。
  - 确认 Bot 拥有 **Manage Messages** 权限。

- **议诉归档失败或发不出文件？**
  - 确认已在 `/议诉 设置` 中指定“归档频道”。
  - 确认 Bot 在归档频道拥有发送消息与发送文件权限。

- **监察归档会暴露投票人吗？**
  - 不会。监察投票仍按匿名规则处理，归档摘要只记录 yes/no/未投票统计，不导出投票人和票值的对应关系。
- **监察归档是否必须删除临时讨论频道？**
  - 不必须。默认是“仅归档”，频道继续保留；需要清理时再手动选择锁定或删除。
