# Discord Odysseia-Court Bot

Odysseia-Court 是一个面向 Discord 服务器治理的中文 Slash Command Bot，提供 **议诉处理**、**监察组案件处理** 与 **公示统一募选** 三组功能，帮助服务器把投诉、裁决、候补监察、匿名投票、岗位募选等流程集中到 Discord 内完成。

> 说明：本项目是服务器治理与流程协助工具，不构成任何法律建议。
> 隐私政策：见 [PRIVACY.md](PRIVACY.md)。

---

## 功能概览

### 议诉系统（`/议诉`）

用于处理服务器成员之间的投诉与裁决流程。

- **议诉申请**：成员可通过 `/议诉 申请` 或入口面板提交议诉申请，填写投诉说明、相关规则、证据链接与附件。
- **入口面板**：管理可发布带“提交议诉申请”按钮的固定入口，方便成员发起议诉。
- **管理审核**：申请提交后进入审核频道，管理可通过、驳回或要求补充材料。
- **公开 / 私密议诉**：申请人可选择公开或私密模式，管理审核时也可覆盖选择。
- **一案一频道**：审核通过后自动创建独立议诉频道，便于集中讨论与权限隔离。
- **回合制发言**：双方按轮次发言，当前发言方需点击“获取本轮发言权”，每轮最多 10 条消息，限时 10 分钟。
- **继续议诉确认**：第 3 轮结束后，双方选择是否继续；双方都同意才进入后续轮次，否则进入裁决。
- **管理裁决**：管理在裁决频道选择“成立并说明”或“不成立并说明”，并填写裁决理由与可选处置。
- **补充证据**：议诉提交后或进行中，可继续补充证据链接与附件。
- **撤诉与归档**：投诉人可撤诉；结案或撤诉后，管理可导出归档并删除议诉频道。
- **审计频道**：可选配置审计频道，记录关键管理操作。

### 公示统一募选系统（`/募选`）

用于在服务器内发起岗位募选、报名、公示、统一投票与结果发布。

- **创建募选**：管理可设置募选名称、岗位与当选人数、报名时间、投票时间、频道与身份组限制。
- **多岗位报名**：候选人通过报名入口选择一个或多个岗位，并填写参选宣言。
- **报名编辑与撤回**：报名期内候选人可查看、编辑或撤回自己的报名。
- **报名身份组限制**：可限制只有指定身份组成员能够报名；未设置时默认所有成员可报名。
- **投票身份组限制**：可限制只有指定身份组成员能够投票；未设置时默认所有成员可投票。
- **实时公示 / 统一公示**：候选人资料可报名后实时公示，也可在报名结束后统一公示。
- **统一投票面板**：投票阶段发布统一投票入口，投票者使用私密分页界面选择候选人。
- **匿名投票明细**：不会公开“谁投给谁”，只发布统计与最终结果。
- **投票自查与运维**：成员可私密查看自己的投票提交情况；管理可查看状态、手动开始投票、结束计票、重算结果、清除异常投票、取消募选或查看审计日志。
- **常态申请投票**：管理可创建长期申请入口，成员按岗位提交宣言后触发单人同意/反对投票或支持票收集；通过、未通过、退出等动态发布到公示频道。

### 监察组系统（`/监察`）

用于管理监察候补、创建监察案件、抽取临时监察组、匿名投票与结果归档。

- **监察候补管理**：管理可添加、移除、查看候补；候补可主动退出。
- **留任确认**：可设置候补留任确认周期，提醒候补确认是否继续留任。
- **启动监察案件**：管理填写案件说明、双方说明、材料链接与响应期限后，Bot 向候补发送参与邀请。
- **候补响应**：候补可通过按钮选择愿意参与或拒绝参与。
- **Ban 与抽取**：响应结束后，可按案件情况进行 Ban 并抽取，最终临时监察组人数为 3 / 5 / 7。
- **临时讨论频道**：抽取完成后创建私密讨论频道，仅相关管理与临时监察成员可见。
- **补抽与取消**：管理可查看案件状态、补抽替换成员或取消案件。
- **匿名投票**：临时监察成员在讨论频道内发起匿名投票，选择“诉求合理”或“诉求不合理”。
- **结果公示**：投票结束后发布匿名统计结果与结论，不公开个人投票明细。
- **监察归档**：已公示裁决或已取消案件可导出归档；归档后可选择保留、锁定或删除临时讨论频道。

## Discord 侧准备

### 1. 创建并邀请 Bot

在 <https://discord.com/developers> 创建应用并添加 Bot，然后邀请进服务器。

建议邀请时勾选：

- Scopes：`bot`、`applications.commands`
- Permissions 建议至少包含：
  - View Channels
  - Read Message History
  - Send Messages
  - Embed Links
  - Attach Files
  - Manage Channels
  - Manage Messages
  - Manage Roles（监察候补身份组管理需要）

### 2. 开启 Privileged Intents

在 Discord Developer Portal 的 Bot 设置中开启：

- **Message Content Intent**

不需要开启 **Server Members Intent**。本项目不监听成员加入/离开事件，也不拉取完整成员列表；成员与身份组信息通过交互对象或按需查询获取。

公开部署前，请将 [PRIVACY.md](PRIVACY.md) 发布到公开可访问地址，并在 Discord Developer Portal 填写 Privacy Policy URL。

### 3. 准备服务器频道与身份组

建议提前准备：

#### 议诉系统

- 管理身份组
- 审核频道
- 议诉频道分类
- 裁决频道
- 归档频道
- 可选：公开议诉观众身份组
- 可选：审计日志频道

#### 监察组系统

- 监察候补身份组
- 管理通知频道
- 临时讨论频道分类
- 裁决公示频道
- 可选：监察归档频道

#### 募选系统

- 报名频道
- 投票频道
- 公示频道
- 可选：告警频道
- 可选：允许报名身份组
- 可选：允许投票身份组

## 本地运行

### 1. 安装依赖

```powershell
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env` 并填写：

```env
DISCORD_TOKEN=
COMMAND_GUILD_IDS=
COMMAND_GUILD_ID=
DB_PATH=data/court.db
BOT_MAX_MESSAGE_CACHE=200
ARCHIVE_CONCURRENCY=1
ARCHIVE_MEDIA_BUDGET_MB=0
ARCHIVE_SINGLE_IMAGE_MAX_MB=0
```

字段说明：

- `DISCORD_TOKEN`：Discord Bot Token，必填。
- `COMMAND_GUILD_IDS`：可选。填写一个或多个服务器 ID 后，指令会快速同步到这些服务器。
- `COMMAND_GUILD_ID`：可选。兼容单服务器旧配置。
- `DB_PATH`：主数据库路径，可选，默认 `data/court.db`。
- `BOT_MAX_MESSAGE_CACHE`：消息缓存数量，可选。
- `ARCHIVE_CONCURRENCY`：同时归档数量，可选。
- `ARCHIVE_MEDIA_BUDGET_MB`：归档媒体总预算，可选；`0` 表示不限制。
- `ARCHIVE_SINGLE_IMAGE_MAX_MB`：单张图片归档上限，可选；`0` 表示不限制。

> 频道、分类、身份组不需要写入 `.env`，请在 Discord 内使用 `/议诉 设置`、`/监察 设置` 与 `/募选 创建` 等指令配置。

### 3. 启动 Bot

```powershell
python main.py
```

## 指令与使用方式

### 议诉系统

#### `/议诉 设置`（管理）

初始化或更新议诉系统配置。

需要选择：

- 管理身份组
- 审核频道
- 议诉分类
- 裁决频道
- 归档频道
- 可选：观众身份组
- 可选：审计频道

#### `/议诉 入口`（管理）

在当前频道发布议诉申请入口面板。成员可点击按钮提交议诉申请。

#### `/议诉 申请`

提交议诉申请。

需要选择或填写：

- 被投诉人
- 议诉模式：公开 / 私密
- 可选：证据链接
- 可选：证据附件 1～3

提交后会弹出表单继续填写申请说明。

#### `/议诉 补充证据`

为议诉补充证据。

可填写：

- 议诉编号：在对应议诉频道内使用时可不填
- 可选：证据链接
- 可选：证据附件 1～3

#### `/议诉 查看设置`

查看当前服务器的议诉系统配置。

### 议诉典型流程

1. 管理执行 `/议诉 设置` 完成初始化。
2. 管理在议诉区执行 `/议诉 入口` 发布申请入口。
3. 成员点击入口按钮或执行 `/议诉 申请` 提交申请。
4. 管理在审核频道处理申请：通过、驳回或要求补充材料。
5. 审核通过后进入独立议诉频道。
6. 双方按回合点击“获取本轮发言权”并发言。
7. 第 3 轮结束后，双方选择是否继续议诉。
8. 进入裁决阶段后，管理发布裁决结果。
9. 结案或撤诉后，管理执行归档并清理频道。

## 募选系统

### `/募选 设置`（Manage Guild 或 Administrator）

查看或更新募选模块管理身份组。未配置时，募选管理命令使用 Discord 原生 `Manage Guild` 或 `Administrator` 兜底；配置后，管理命令允许 `Administrator` 或任一募选管理身份组使用。

参数：

- 可选：管理身份组 ID/提及，逗号或空格分隔。
- 填 `清空`、`none`、`all` 可清空设置并恢复原生权限兜底。

### `/募选 创建`（管理）

创建一场募选。

主要参数：

- 名称
- 岗位配置，例如：`大当家:1,二当家:3,执行成员:9`
- 每人最多投票数
- 公示模式：实时公示 / 报名结束后统一公示
- 报名持续时间，例如：`3天`、`72小时`、`2天6小时`
- 公示持续时间，例如：`1天`、`12小时`、`0小时`
- 投票持续时间，例如：`2天`、`48小时`
- 报名频道
- 投票频道
- 公示频道
- 可选：允许报名身份组
- 可选：允许投票身份组
- 可选：告警频道
- 可选：报名开始时间，格式为北京时间 `YYYY-MM-DD HH:mm`
- 可选：是否立即发送报名入口

### 常用成员操作

- 报名入口按钮：报名、查看我的报名、编辑报名、撤回报名。
- `/募选 我的报名`：查看自己的报名记录。
- 投票面板按钮：开始投票、翻页选择、完成选择、确认投票、查看我的投票。
- `/募选 我的投票`：私密查看自己是否已投票、已提交的选择或作废状态；`/募选 状态`：查看募选状态。
- `/募选 列表`：列出服务器内募选。
- `/募选 帮助`：查看募选系统内置帮助。

### 常用管理操作

- `/募选 设置`：查看或更新募选模块管理身份组。
- `/募选 设置入口`：发送或重发报名入口。
- `/募选 候选人`：查看、打回、撤销、恢复或重发候选人公示。
- `/募选 报名身份组`：查看或更新允许报名身份组。
- `/募选 投票身份组`：查看或更新允许投票身份组。
- `/募选 同步公示`：修复或刷新候选人公示。
- `/募选 开始投票`：应急手动开始投票。
- `/募选 结束并计票`：应急手动结束并发布结果。
- `/募选 计票预览`：只预览计票结果，不发布。
- `/募选 重算结果`：重新计算并发布结果。
- `/募选 清除投票`：作废某位成员的投票记录。
- `/募选 取消`：取消未完成募选。
- `/募选 审计`：查看募选操作日志。
- `/募选 运维自检`：检查未完成募选的状态、频道与权限。
- `/募选 运维tick`：手动执行一次阶段检查。
- `/募选 运维推进`：强制推进指定募选到下一阶段。

### 募选典型流程

1. 管理执行 `/募选 创建`，填写岗位、时间、频道与权限限制。
2. 如未自动发送入口，管理执行 `/募选 设置入口`。
3. 成员在报名入口点击“报名”，选择岗位并填写参选宣言。
4. 报名期内，成员可查看、编辑或撤回报名。
5. 报名结束后，根据公示模式完成实时公示或统一公示。
6. 投票阶段开始后，成员在投票面板点击“开始投票”。
7. 投票者在私密界面选择候选人并确认提交。
8. 投票结束后发布结果。
9. 如需要修正，管理可使用计票预览、重算结果、清除投票等管理指令。

### 常态申请投票（`/募选 常态`）

用于长期开放单人申请。成员一次选择一个岗位并填写报名宣言，Bot 会在投票频道发布一条包含申请信息与投票/支持按钮的一体式消息。

管理创建：

- `/募选 常态 创建`
- 必填：名称、岗位列表、入口频道、投票频道、公示频道。
- 可选模式：同意/反对投票（默认）、支持票收集。
- 同意/反对投票规则：最低总票数（默认 1）、同意比例百分比（默认 51）、投票时长（默认 7天）、冷却期（默认 7天）。
- 支持票收集规则：支持目标票数（必填，达到后立即通过并公示支持者）、投票时长（默认 7天）、冷却期（默认 7天）。
- 可选限制：允许申请身份组、允许投票身份组；不填均表示所有成员可用。是否立即发送入口默认开启。

成员流程：

1. 在长期入口点击“提交申请”，选择一个岗位并填写报名宣言。
2. 同意/反对投票模式：投票期间不显示实时票数；有投票资格的成员可投同意/反对，并可在结束前改票。
3. 支持票收集模式：收集期间不显示实时票数；有支持资格的成员可支持或撤回支持。支持票达到目标后立即通过，并公示全部支持者。
4. 到期后自动结算：同意/反对投票按总票数和同意比例判定；支持票收集未达目标则未通过，投票面板和公示频道只显示未通过，不公示支持者名单和支持票统计。
5. 未通过、进行中主动退出、通过后退出名单，都会进入配置的冷却期。
6. 已通过且未退出的成员不能重复申请同一配置。

常态入口按钮：

- `提交申请`：提交一个岗位申请。
- `我的申请`：私密查看自己的最近申请、结果和冷却状态。
- `退出申请`：二次确认后退出进行中的申请，或将自己移出通过名单。

常态管理与查询：

- `/募选 常态 入口`：发送或重发长期入口。
- `/募选 常态 状态`：查看常态配置、进行中申请数和通过人数。
- `/募选 常态 通过名单`：公开查看通过名单，可按配置 ID 和岗位过滤。
- `/募选 常态 结束投票`：手动结算指定申请。
- `/募选 常态 取消申请`：管理员取消投票中的申请。
- `/募选 常态 移除通过`：管理员将成员移出通过名单，并写入冷却期。

## 常见问题

### 看不到 Slash Command？

- 确认 Bot 已使用 `applications.commands` scope 邀请。
- 若使用 `COMMAND_GUILD_IDS`，确认填写的是正确服务器 ID。
- 重启 Bot 后等待同步完成，并在 Discord 客户端按 `Ctrl + R` 刷新。

### 按钮点击后没有反应？

- 确认 Bot 在线。
- 确认 Bot 在相关频道拥有查看、发言、嵌入链接、发送附件等权限。
- 涉及创建或删除频道时，确认 Bot 拥有 Manage Channels 权限。

### 回合发言限制不生效？

- 确认已开启 Message Content Intent。
- 确认 Bot 拥有 Manage Messages 权限。

### 议诉公开模式无法创建？

- 公开议诉需要配置观众身份组。
- 请使用 `/议诉 设置` 选择观众身份组后再通过公开议诉申请。

### 募选无法报名或无法投票？

- 确认募选当前处于正确阶段。
- 如果配置了报名或投票身份组，确认成员拥有对应身份组之一。
- 管理可使用 `/募选 状态` 或 `/募选 运维自检` 查看当前配置。

### 监察投票会公开投票人吗？

不会。监察投票结果只公示匿名统计与结论，不公开个人投票明细。

### 归档是否一定会删除频道？

不一定。议诉结案时可归档并删除议诉频道；监察归档可选择仅归档、归档并锁定频道或归档并删除频道。
