# Odysseia-Court 隐私政策

最后更新：2026-06-12

本隐私政策适用于 Odysseia-Court Discord Bot（下称“本 Bot”）。本 Bot 用于 Discord 服务器内的议诉处理、监察案件、候补监察、募选报名、匿名投票、结果公示、审计与归档。

如果你自行部署本项目，你是该部署实例的数据控制者。公开使用前，请将本政策中的联系方式和留存安排更新为你的实际运营信息，并在 Discord Developer Portal 中填写公开可访问的隐私政策链接。

## 我们收集的数据

本 Bot 只在提供服务器治理功能所需范围内处理数据，可能包括：

- Discord 标识符：用户 ID、服务器 ID、频道 ID、消息 ID、身份组 ID。
- 成员信息：用户名、显示名、头像 URL、身份组和权限信息。通常来自用户交互 payload 或按已知用户 ID 查询。
- 议诉数据：投诉人、被投诉人、申请说明、涉及规则、证据链接、附件元数据、审核与裁决记录。
- 监察数据：案件说明、双方陈述、监察候补名单、响应记录、抽取/补抽记录、匿名投票统计与案件归档记录。
- 募选数据：报名岗位、参选宣言、候选人状态、投票记录、作废记录、计票结果和审计日志。
- 消息内容：仅在功能需要时读取本 Bot 可访问频道中的消息内容、附件、嵌入内容和组件信息，主要用于议诉/监察临时频道的发言监管、案件复盘和 HTML/ZIP 归档。
- 运行数据：错误信息、操作日志、配置项和数据库记录，用于维护、排错和审计。

本 Bot 不请求 Server Members privileged intent，不监听成员加入/离开事件，也不拉取完整服务器成员列表。

## 我们如何使用数据

收集的数据用于：

- 创建、审核、推进和结束议诉或监察案件。
- 管理候补监察成员、案件响应、抽取、补抽和匿名投票。
- 管理募选报名、公示、投票、计票和异常投票处理。
- 检查身份组、权限和频道访问控制。
- 生成管理可见的案件归档、审计记录和故障排查信息。
- 防止流程滥用、误操作和重复投票。

本 Bot 不会将数据用于广告投放、用户画像、出售、出租、跨服务器营销，或训练通用 AI 模型。

## 消息内容的处理

本 Bot 的 Message Content 访问主要服务于案件和归档功能：

- 在议诉频道中，本 Bot 会监听消息事件以执行回合发言限制、无权限发言删除和发言计数。
- 管理触发归档时，本 Bot 会读取对应议诉频道或监察临时讨论频道的历史消息，并生成 HTML/ZIP 归档文件。
- 归档可能包含消息正文、附件链接、图片附件、嵌入内容、发送者显示名和发送时间。
- 归档文件会发送到服务器配置的归档频道或管理通知频道，其后续可见性取决于服务器频道权限设置。

本 Bot 不会为了无关目的持续扫描所有频道消息。只有本 Bot 已被授予访问权限的频道数据才可能被处理。

## 数据共享

本 Bot 不会主动向第三方出售或出租数据。

数据可能在以下情况下被共享或可见：

- Discord 平台本身会处理消息、交互、附件和用户资料数据。
- 服务器管理员、裁决人员或被授权身份组可在配置频道内查看审核、裁决、归档和审计记录。
- 归档文件发送到配置频道后，拥有该频道访问权限的成员可能看到归档内容。
- 如为遵守适用法律、Discord 要求或处理安全事件，运营者可能需要披露必要信息。

## 数据存储与留存

本 Bot 默认使用部署环境中的 SQLite 数据库保存配置、案件、报名、投票和审计记录。

- 数据保存位置由部署者配置，默认在本 Bot 的运行目录或 `data/` 目录下。
- 案件、审计、报名、投票和归档记录通常会保留，直到服务器管理员或 Bot 运营者删除、清理或重置相关数据库。
- 归档文件的留存由归档频道、服务器管理策略和 Discord 平台的消息留存能力决定。
- 如果频道被删除，Discord 中的原始频道消息可能不可再访问，但本 Bot 已生成的归档文件和数据库记录可能仍然存在。

## 数据删除与访问请求

你可以请求访问、更正或删除与自己相关的数据。请通过以下方式联系：

- 如果你是普通成员：联系安装本 Bot 的 Discord 服务器管理员。
- 如果你是服务器管理员：联系本 Bot 的运营者或维护者，并提供服务器 ID、用户 ID、相关案件/募选 ID 和请求内容。

运营者会在合理范围内处理请求。对于服务器治理、审计、安全或争议处理所必需的记录，运营者可能选择保留必要记录、限制访问、脱敏或删除部分字段，而不是立即删除全部记录。

## 安全措施

本 Bot 通过 Discord 频道权限、身份组权限、私密响应、审计日志和本地数据库访问控制来限制数据访问。部署者应当：

- 妥善保护 Bot token、数据库文件、服务器主机和备份。
- 只给 Bot 授予运行所需的最小权限。
- 将归档频道、审核频道、裁决频道和监察临时频道设置为仅授权人员可见。
- 定期检查部署环境、依赖和 Discord Developer Portal 配置。

## 儿童与未成年人

本 Bot 面向遵守 Discord 服务条款的服务器成员使用。运营者不会故意收集 Discord 不允许使用其服务的用户数据。服务器管理员应根据 Discord 的条款和本服务器规则管理成员访问。

## 政策变更

如果本 Bot 的数据处理方式发生重大变化，本政策会更新“最后更新”日期，并通过仓库、支持渠道或服务器公告等方式公布。

## 联系方式

请通过本 Bot 所在 Discord 服务器的管理渠道、Bot 的公开支持渠道，或本项目公开仓库的 Issue/联系方式联系运营者。
